# Content Repurposing Guide

This reference covers strategies for extending the life and reach of content by adapting it across formats, platforms, and audiences. Read this when the user has existing content and wants to get more value from it without starting from scratch.

**The principle:** Repurposing isn't recycling. Every format change should serve a different audience need, a different consumption context, or a different stage of awareness. If the new format doesn't add value, don't repurpose.

---

## 1. When to Repurpose (and When Not To)

### Good reasons to repurpose
- The original piece has evergreen value and is performing well.
- The insight deserves to reach audiences on other platforms.
- The original format doesn't match how some readers prefer to consume (video vs text, short vs long).
- You need to fill a content calendar without starting from scratch.
- You want to reinforce a message across multiple touchpoints.
- The original content can be disaggregated into smaller, more actionable pieces.

### Bad reasons to repurpose
- You're out of ideas and need to publish something.
- The original piece was weak and no format change will fix it.
- You're trying to rank for the same keyword with similar content on two pages (cannibalization risk).
- The content is time-sensitive and past its relevance window.
- You're reformatting without adapting the voice or angle for the new medium.

## 2. The Repurposing Matrix

Match source format to target format based on the reader's consumption context and the core insight's adaptability.

### Source → Target matrix

| Source format | Repurpose into |
|---|---|
| **Long-form blog post / pillar page** | Twitter thread, LinkedIn carousel, email newsletter, video script, podcast outline, infographic, slide deck, checklist/gated asset, 3-5 social posts, guest post excerpt |
| **Video / webinar** | Blog post (transcript + structure), audio-only podcast, LinkedIn post (5 key takeaways), Twitter thread (timestamps), email recap, quote cards (images), tutorial steps |
| **Podcast** | Blog post (edited transcript), social posts (best quotes), YouTube clips, newsletter summary, quote cards, show notes as blog |
| **Case study** | Landing page, one-pager PDF, sales deck slide, social post (customer quote), email case study, video testimonial script, comparison page evidence block |
| **Email newsletter** | Blog post (if the topic has long-term relevance), social posts (best line), thread starter, podcast topic |
| **Research / data report** | Infographic, blog highlights, social stat cards, press release, webinar data deep-dive, tweetstorm of key findings, slide deck, gated summary PDF |
| **Webinar / event talk** | Blog post (transcript + narrative), video clips (per topic), podcast episode, social posts (per stat), email recap, checklist, slide deck |
| **Social post / thread** | Blog post (expanded), newsletter item, video topic, case study evidence, comparison piece |
| **FAQ / knowledge base** | Blog post (problem-solution), social tip series, video short (one Q per video), email tip |
| **Product changelog** | Social announcement, email highlight, blog post (major releases), video demo |

## 3. The Repurposing Workflow

### Step 1: Evaluate the source
- Is the content evergreen or timely? (Evergreen = more repurposing options.)
- What's the core insight that travels across formats?
- What format-specific value would the source lose if just clipped? (A blog post's nuance may not fit a tweet. A video's visual proof doesn't translate to audio.)

### Step 2: Choose target formats by audience need
| Audience need | Best format |
|---|---|
| "Give me the highlights fast" | Twitter thread, LinkedIn post, email summary |
| "I want to understand deeply" | Blog post, video, podcast |
| "I need to reference this later" | Checklist, PDF, infographic, saved thread |
| "I want to be convinced" | Case study, testimonial, data visualization |
| "I learn by watching" | Video, slide deck, visual tutorial |

### Step 3: Adapt, don't clip
- Each format has its own job. Don't just cut and paste.
- A blog post → Twitter thread: rewrite the key insight as a hook, then unpack with one tweet per sub-point.
- A video → blog post: don't just transcribe. Structure for reading (headings, short paragraphs, scannability).
- A case study → social post: lead with the result, not the backstory.

### Step 4: Add format-specific value
Every repurposed piece should add something the original didn't:
- Blog post → LinkedIn: add a personal reflection or recent example.
- Podcast → blog: add headings, links, and screenshots.
- Webinar → email: add a "since the webinar" update.

### Step 5: Cross-link
- The new piece should link to the original (and vice versa, when relevant).
- This signals to search engines that the content is part of a broader resource.
- It also gives interested readers a path to deeper content.

## 4. Format-Specific Repurposing Playbooks

### Blog post → Twitter/X thread
```
Tweet 1: Hook. The one surprising or useful line. (Scroll-stopper.)
Tweet 2: The problem or context. Why this matters.
Tweet 3: Key insight #1 (punchy, standalone).
Tweet 4: Key insight #2.
Tweet 5: Key insight #3 (or a numbered list).
Tweet 6: Example or anecdote (1 tweet).
Tweet 7: The takeaway or CTA ("Full piece here: link").
```
**Length:** 5-10 tweets max. Once per sub-topic.
**Adaptations:** Use punchier language. Remove transitions. Add emoji for scannability.

### Blog post → LinkedIn post
```
[First 2 lines: must earn "see more". A bold claim, a question, a counterintuitive line.]

[Body: 3-5 short paragraphs. 1-3 sentences each. Line breaks between every paragraph.
 Keep the original insight but rewrite in first person. Add a "here's what I learned" frame.]

[Takeaway: 1 line the reader can act on.]

[Question: invites comments.]
```
**Adaptations:** More personal framing than the blog. Add a "full post in comments / first comment" link.

### Blog post → Email newsletter
```
SUBJECT: [The hook: shorter than blog headline]
PREHEADER: [The benefit: what they'll learn]

[Opening: connect to previous email or current events, 1-2 sentences.]

[Body: 2-3 short paragraphs summarizing the key insight. Not the whole post.
 Front-load the value. Include the link early.]

[PS: One extra tip or resource not in the original piece.]
```
**Adaptations:** The email should add value beyond the link. Include an exclusive takeaway or an additional resource.

### Video / webinar → Blog post
```
HEADLINE: [Searchable. Include the main takeaway.]

[Intro: 2-3 sentences. What the video covered and why it matters.]

[Section 1: Key insight from the first segment.]
[Section 2: Key insight from the second segment.]
[Section 3: Key insight from the third segment.]

[Key quotes: best 2-3 lines from the video, attributed.]

[Where to watch: link to the full video for context.]

[Takeaway / CTA]
```
**Adaptations:** This is NOT a transcript. Select and structure. Add context that wasn't in the video. Use headings for scannability.

### Case study → Social proof block (for landing page)
- Pull the single most impressive metric: "Acme cut support response time 65%."
- Write a one-sentence version: "Acme Corp, a 200-person fintech, cut support response time from 45 minutes to 8 minutes with Acme AI."
- For testimonials, pull the most quotable line from the customer quote.
- For comparison pages, pull one specific differentiator the case study demonstrates.

### Long report → Infographic / visual summary
- Extract 5-7 key statistics or findings.
- Order them from most surprising to most actionable.
- Write a one-line takeaway per stat.
- Design as a vertical scroll (social-native) or single image (for embedding).

## 5. The Content Atomization Model

Break one piece of content into increasingly smaller units:

```
                    ┌──────────────────────┐
                    │   PILLAR CONTENT      │ (5,000 words)
                    │  The comprehensive     │
                    │  guide / report        │
                    └──────────┬───────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
         ┌────┴────┐    ┌─────┴─────┐    ┌────┴────┐
         │  BLOG   │    │  VIDEO    │    │ PODCAST │
         │ POST 1  │    │ (summary) │    │ (deep)  │
         └────┬────┘    └─────┬─────┘    └────┬────┘
              │               │               │
         ┌────┴────┐    ┌────┴────┐    ┌─────┴─────┐
         │ EMAIL   │    │ SOCIAL  │    │ QUOTE     │
         │ SUMMARY │    │ THREAD  │    │ CARDS     │
         └─────────┘    └─────────┘    └───────────┘
```

### Atomization rules
- Each level should serve a different consumption context.
- Don't release all at once: space them out over days/weeks.
- The smallest piece (quote card) should still pass the "would I share this?" test on its own.
- Link smaller pieces back to the parent piece.

## 6. Repurposing by Platform Context

| Platform | Consumption context | Adapt repurposed content by |
|---|---|---|
| **Blog** | Search, focused reading, reference | Headings, depth, links, scannability, SEO optimization |
| **Twitter/X** | Scrolling, quick hits, conversation | Punchy hook, tight threads, one idea per tweet |
| **LinkedIn** | Professional scrolling, credibility-seeking | First-person framing, longer form, professional insight |
| **Email** | Inbox, action-oriented | Short, scannable, one CTA, value-first |
| **YouTube** | Active watching, learning | Visual proof, storytelling, clear structure |
| **Podcast** | Passive listening, multitasking | Audio-first: conversational, descriptive, no visual dependencies |
| **Instagram** | Visual browsing, inspiration | Hook in image/text overlay, short caption, aesthetic format |
| **SlideShare / decks** | Research, professional reference | Self-contained slides, visual hierarchy, conclusion-recommendation |
| **Newsletter** | Curated reading, trust-based | Personal voice, exclusive angle, direct relationship |

## 7. Repurposing Calendar Strategy

Instead of repurposing all at once, sequence pieces to build momentum:

### Week 1: Publish the original (pillar)
Publish the full piece on your blog or primary platform.

### Week 2: Extract the highlights
- Social thread (Twitter/X or LinkedIn)
- Email summary to your list

### Week 3: Adapt for a different format
- Video (summary or deep-dive on one section)
- Podcast episode (if you have one)

### Week 4: Make it visual
- Quote cards from the best lines
- Infographic of key data points
- Carousel post for LinkedIn/Instagram

### Week 5: Syndicate or guest post
- Republish an adapted version on Medium, Dev.to, or industry publication
- Include a canonical link back to the original

### Week 6: Refresh and re-share
- Update with new data or examples
- Re-share on social with a new opening line

## 8. Repurposing Pitfalls

| Pitfall | Why it fails | Fix |
|---|---|---|
| Copy-paste across platforms | Platform-native expectations are violated | Rewrite for each platform's conventions |
| No added value | "I already saw this" | Add format-specific value (context, update, personal take) |
| Wrong format for the insight | A data-heavy insight gets turned into audio where listeners can't see the numbers | Pick formats where the core insight shines |
| Over-repurposing | The same insight across 5 channels in 2 days feels spammy | Space out repurposed pieces. 1-2 per week from the same source |
| No cross-linking | Reader can't find the original | Link smaller pieces back to the parent piece |
| Format mismatch | A 60-minute webinar becomes a 5,000-word blog post with no editing | Cut ruthlessly. The blog should be tighter, not longer |
| Ignoring performance data | Repurposing content that didn't resonate | Only repurpose content with proven engagement or evergreen value |
| SEO cannibalization | Two pages targeting the same keyword compete | Use canonical tags, different angles, or different target keywords |
