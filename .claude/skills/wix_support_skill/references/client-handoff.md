# Client Handoff & Collaboration — Reference

## Collaborator Roles

**Add:** Dashboard → Settings → Roles & Permissions → (+) Invite People

| Role | Access Level |
|------|-------------|
| Website Editor | Can edit site in editor |
| Admin | Full dashboard + editor |
| CMS Editor | CMS content only (no design) |
| Blog Writer | Create/edit own posts |
| Blog Editor | Create/edit all posts |
| Store Manager | Orders, products (no editor) |
| Bookings Manager | Bookings calendar (no editor) |
| SEO Manager | SEO settings |
| Customer Service Agent | View orders, contact customers |
| Analytics Manager | View analytics only |

> **Best practice:** Give clients minimum permissions. Most clients should be "CMS Editor" or "Blog Writer" — not "Admin."

## Transferring Site Ownership
**Path:** Dashboard → Settings → Account → Transfer Primary Ownership
1. Enter new owner's Wix email address
2. New owner receives email to accept transfer
3. Once accepted, they own the site + billing
4. Old owner remains as Admin unless they remove themselves

## Client Training Checklist
Before handoff, train clients on:
- [ ] Accessing the Wix Dashboard (not the editor, unless needed)
- [ ] Adding/editing blog posts
- [ ] Updating CMS content (products, team, etc.)
- [ ] Viewing orders and managing fulfillment
- [ ] Responding via Wix Inbox
- [ ] Checking analytics
- [ ] Adding contacts and sending emails
- [ ] What to NOT change (complex design/Velo sections)

## Protecting Your Work
- **Classic Editor:** Lock elements → right-click → Lock Position
- **Velo areas:** Add code that prevents CMS from affecting sections
- **Editor warning:** Add a box with "DO NOT EDIT BELOW THIS LINE" (visible in editor, hidden on live)

## Documentation Best Practices
Create a client handoff document including:
1. **Login details** — how to access dashboard
2. **What they can edit** — specific pages, CMS collections, blog
3. **What to avoid** — Velo sections, header/footer structure
4. **Key contacts** — who to call for technical issues
5. **Maintenance tasks** — backups, plan/domain renewal dates
6. **Change request process** — how to request edits from you

## Site Duplication (For Agencies)
Dashboard → Site → Manage Site → More Options → Duplicate Site
Creates a standalone copy. Good for: client templates, A/B testing, backup before major changes
